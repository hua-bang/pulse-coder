Agent 开发记录和感想

一图总结 Pulse Agent 从好奇心驱动的 Demo，到可扩展工程化体系，再到自举迭代与深度反思。

关键词：开放 · 迭代 · 频率

开发历程（从 Demo 到自举）

好奇心驱动：为理解 Agent Loop 与上下文组织，研究开源框架。

最小 Demo：写出第一版 Coder，体验工具调用循环。

工程化迁移：多包架构、基础能力完善，构建可维护基建。

插件机制：支持 MCP/Skill/Task 等机制可插拔扩展。

远程协作：接入飞书/Discord，实现异步协作。

自我迭代：Agent 参与 PR 与代码演进，自举迭代。

定位与能力

CLI 入口简版 Coding CLI，命令与会话管理。
飞书/Discord远程对话协作与任务执行。
Agent 套件SDK + 工具机制 + 插件生态。
基础运行时层：Agent Loop + 工具调度
机制/能力层：MCP / Skill / SubAgent / Memory
入口层：CLI / Discord / Feishu / Web（可扩展）

技术实现要点

Agent Loop：感知 → 决策 → 执行 → 反馈 → 终止
插件钩子：beforeRun / beforeLLM / tool hooks / afterRun
机制注入：工具化 + System Prompt 动态拼接
通道适配：PlatformAdapter 统一输入/输出
“将上下文与工具调用原文交给模型，自主判断执行动作与完成任务。”

开发感想

信息处理频率：AI 让信息流动更快，也带来压缩。
可持续迭代：白盒可控比黑盒更重要。
时间感改变：并行 Agent 让“时间”被重新定义。
人与工具：彼此塑造，Human in the loop === AI in the loop。
实践场：抽象理论需体感验证，构建自己的试验场。
收束观点 保持开放谦逊、建立反馈迭代机制、适配时代频率。
