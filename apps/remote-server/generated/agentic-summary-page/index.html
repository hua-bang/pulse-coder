AGENTIC ENGINEERING BRIEF · 2026

AI 编程正在经历范式转移：
从“手工编码”走向“软件工厂编排”

基于 Karpathy 与 Michael Truell 的观点：开发已从逐行输入代码，升级为让多个 Agent 在云端并行执行。 人类职责从“写细节”转向“定义问题、设约束、做验收与纠偏”。

核心结论：抽象层上移，但细节责任没有消失，只是从“实现细节”迁移到“管理细节”。

Karpathy：阶段突变

关键词：非线性跃迁、长程一致性、任务接管

  • 2025 年 12 月前后，coding agent 从“基本不可用”跨到“可独立完成长任务”。
  • 开发流程被重写：人类在英文层面分派任务，再并行审查结果。
  • 高杠杆来自构建 orchestrator：工具、记忆、指令与多 Agent 协同。
  • 仍需判断与监督：方向、品味、边界 case、测试验收不可省略。

Truell（Cursor）：第三时代

关键词：云端自治、工厂化、并行规模化

  • 第一阶段 Tab：逐行提效；第二阶段同步 Agent：高频人机回合。
  • 第三阶段是云端自治 Agent：更长时、更大任务、低监督执行。
  • 关键数据：Agent 使用量一年 15x+，内部约 35% PR 已由自治 Agent 生成。
  • 角色变化：开发者从编码者，转向软件生产系统编排者。

你问到的关键：还要不要懂细节？

答案：要。不是“全懂再做”，而是“识别关键细节并可验证”。

1. 目标
明确要达成的业务结果
2. 约束
成本、时延、安全、兼容性
3. 边界
失败样例与极端输入
4. 验收
可跑测试与通过标准
